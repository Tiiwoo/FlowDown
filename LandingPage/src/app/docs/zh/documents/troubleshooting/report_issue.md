# 问题反馈与支持

我们将优先处理通过以下渠道提交的反馈。为保护隐私，提交前请务必移除密钥、个人隐私数据，截图若包含敏感信息请进行遮挡。

## 1. GitHub Issue（最推荐）

- 仓库地址：[https://github.com/Lakr233/FlowDown/issues](https://github.com/Lakr233/FlowDown/issues)
- 建议包含以下信息：
  - 设置页底部显示的版本号
  - 复现步骤
  - 期望行为与实际结果
  - 复现后获取的日志：设置 → 支持 → 查看日志 → 共享 **请删除敏感信息**
  - 若涉及同步，请说明 **设置 → 数据** 中的 iCloud 同步开关状态和同步范围
- 若能选择标签（bug/request/question），请一并标注。

## 2. Discord 社区

- 加入 [discord.gg/UHKMRyJcgc](https://discord.gg/UHKMRyJcgc) 与社区成员和维护者讨论。
- 适合排查配置问题、对比模型效果、确认服务状态等。

## 3. 邮件支持

- 仅在不便公开的问题（如安全漏洞、商业授权）时使用邮件联系。
- 地址：flowdownapp@qaq.wiki
- 请说明背景、附上脱敏日志（查看日志 → 共享），并注明问题发生时间。

## 报告前请先确认

- 重启应用，关闭 VPN/代理后再次尝试。
- 复现问题后，立即通过 **设置 → 支持 → 查看日志 → 共享** 获取最新日志。
- 若是同步问题，请在 **设置 → 数据** 中确认 iCloud 同步状态和范围。
- 若是 TestFlight 版本，请注明设置页底部的版本号。
