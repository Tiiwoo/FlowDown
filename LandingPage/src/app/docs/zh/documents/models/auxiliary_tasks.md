# 辅助任务模型

辅助任务模型负责在后台处理一些额外工作，比如给对话起标题、生成封面图，或者在不打断主对话的情况下执行轻量级任务。有些任务会在回复前处理，有些在回复后补充。

## 功能概览

- **标题生成**：给对话列表生成简短准确的标题。
- **图标生成**：挑一个跟主题匹配的 emoji 或图案。
- **网页搜索预处理**：开启浏览时，通过 Web Search 工具判断是否需要搜索，并生成最多 3 条查询词；需要模型支持工具调用。
- **视觉预处理**：给没有文字描述的图片附件生成 `[Image Description]`，并附带 OCR/二维码识别结果，方便那些看不懂图片的模型使用。

## 运行时机

- **主回复前**
  - 图片附件缺少 `textRepresentation` 时，会由视觉辅助任务模型来描述；如果开了“尽量跳过视觉任务”且聊天模型本身能看懂图片，就会跳过。
  - 开启浏览时，任务模型会配合 Web Search 工具决定是否需要搜索（`search_required`）并生成查询词；模型必须支持工具调用，否则开启浏览会导致发送失败。
- **主回复后**
  - 只有标记为自动重命名的会话才会刷新标题和图标。手动改过标题/图标，或者在菜单里手动生成过，都会关闭自动重命名；不过你随时可以在菜单里重新生成。

## 配置入口

在 **设置 → 推理** 的默认模型区域管理：

- **使用对话模型**：直接用当前的聊天模型来做辅助任务；关掉后可以单独指定别的模型。
- **任务模型**：用来生成标题/图标和搜索关键词；建议选速度快、成本低的模型，Apple Intelligence 也能用。
- **视觉辅助任务模型**：当主模型看不懂图片时，用这个模型来预处理图片；Apple Intelligence 不支持这个，请选个能看懂图片的模型。
- **尽量跳过视觉任务**：如果主模型已经能看懂图片，就跳过预处理，省点资源（注意：如果后面切到看不懂图片的模型，之前的图片因为没生成过描述，可能就读不了了）。

## 模型使用清单

| 插槽                             | 作用                                          | 说明                                                               |
| -------------------------------- | --------------------------------------------- | ------------------------------------------------------------------ |
| 任务模型                         | 生成对话标题、图标、网页搜索查询              | 开启浏览且没启用工具调用时必须有；优先选速度快、成本低的文本模型。 |
| 视觉辅助任务模型                 | 给看不懂图的模型生成图像描述 + OCR/二维码结果 | 只有聊天模型看不懂图时才需要。                                     |
| 聊天模型（开启“使用对话模型”时） | 兼职做任务模型的工作                          | 直接用当前聊天模型；如果想用专门的轻量模型，就把这个选项关了。     |

文档可能滞后于代码更新，如有差异请以应用内设置说明和当前发行说明为准。

## 使用建议

- 文本类辅助任务最好用轻量模型，响应快还省钱。
- 打算用网页浏览的话，一定要先配好任务模型，不然发不出消息。
- 主模型能看懂图时，保持“尽量跳过视觉任务”开启；如果非要给看不懂图的模型生成确定的图像描述（含 OCR/二维码），就把它关了。
- 如果自动生成的标题/图标不满意，可以换个更强的任务模型，或者在会话菜单里重新生成；手动改了之后就不会自动重命名了。
